# Memory

このディレクトリには、プロジェクト全体で共有される重要な知識とガイドラインを保存します。

## ファイル一覧

- `constitution.md` - プロジェクトの憲法原則（必須遵守の原則）
- `development-guidelines.md` - 開発ガイドラインとベストプラクティス（実装から得られた知見）

## 使用方法

### Constitution（憲法原則）

すべての開発は `constitution.md` に定義された原則に従う必要があります。これらの原則は、技術選定、アーキテクチャ設計、コード品質の基準として機能します。

### Development Guidelines（開発ガイドライン）

`development-guidelines.md` には、プロジェクトの初期実装から得られた重要な知見とベストプラクティスがまとめられています。新しい機能を実装する際の参考として活用してください。

## 更新方針

- **Constitution**: プロジェクトの根本的な原則。変更は慎重に検討し、チーム全体で合意を得る必要があります。
- **Development Guidelines**: 実装から得られた知見を継続的に追加・更新します。重要なパターンやベストプラクティスを記録します。
